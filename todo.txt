# 全分散式 Java 语义抽离混淆器 TodoList

## 架构和混淆规则
- 架构规则：混淆后的代码绝不能使用统一入口点、Core或中央调度器。所有功能必须分散到独立的单一用途的代理类中。
- 代码生成规则：混淆后的业务逻辑类只允许包含数字常量、变量赋值以及对分散的静态代理类的调用。必须严格避免字符串字面量、'new' 关键字、原始名称、控制流关键字(if/for/while/switch)和真实的API调用。
- 命名规范规则：生成的混淆代理类必须严格使用特定的前缀并加上ID：字符串使用 'S'，类实例化使用 'C'，方法调用使用 'M'，字段操作使用 'F'，控制流使用 'IF' 或 'FOR'。
